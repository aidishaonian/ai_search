LUBE
----

This is the folder for all lube scripts, here you will find the following core files

-- root
boot.lua 		- the bootstrap file for starting the menu system. Loaded by app side front end. Start Here!
version.lua 		- autogenerated file for versioning the WAD file.

-- system 
system\auxiliarry.lua	- provides system level support functions (require\release\wait\reload)
system\lube.lua		- core lua definition support functions (lube.require_type, lube.menu_instance, lube.seeall)
system\event.lua	- event id defintions 
system\message.lua	- message id defintions 
system\txt.lua		- text meta lookup to bz localised string data

-- definitions
colour.lua		- basic colour defintions
font.lua		- font defintion & loading
image.lua		- currently string lookup for all images used in game, will be updated to become image loader & management layer
location.lua		- screen anchor point defintions
models.lua		- 3d asset location information 
sounds.lua		- sound definition & load
types.lua		- defines the item types used in menu definitions
user.lua		- commonly used lua functions should be placed here

-- definitions\lube
this directory contains all of the type information for menus, items, pointers and system objects used by the menu defintions
definitions\lube\item 	- all menu item definitions
definitions\lube\menu 	- all menu type definitions
definitions\lube\dialog 	- all dialog template definitions
definitions\lube\pointer 	- definition of pointer objects for pointing input devices
definitions\lube\system 	- ancillary types, control_layers 
definitions\lube\animation	- flash style animation playback support

 
-- definitions\menus
this directory contains all of the menu defintions used by the app
all game menu definitions should go here




Skinning support
----------------

Skinning is achieved by the use of a series of file overides on the common directory structure.
Any new skin should reside in a directory that mirrors the structure of the "common" directory, 
and only those files that need to be overidden should be present within the skin directory.
Within boot.lua the system variable "package.path" defines the directory search order for all 
module scripts loaded using the require statement, this defines the order in which skins will 
be loaded.
It is anticipated that many skins will be able to be defined by over riding the "types.lua" file 
for cosmetic changes such as image and colour redefinition, however the system supports the 
ability to override any script present within the common directory.
